# 记账日历账本和周月年账统计设计

## 背景

日记模块已经使用 `年 / 月 / 周` 日历选择体验。用户希望记账模块也采用同样的时间组织方式：账本像日记一样用日历选日期，统计像日记总结一样按周账、月账、年账查看。

## 目标

- 账本页改为日历账本：通过 `年 / 月 / 周` 日历选择日期，再展示当天账单。
- 新增账单时默认使用账本当前选中的日期。
- 统计页改为 `周账 / 月账 / 年账` 三种模式。
- 周账、月账、年账都展示收入、支出、结余、趋势和分类排行。
- 继续保持记账数据和日记 Markdown、GitHub、`.diary` 分离。

## 非目标

- 不修改日记页日历组件行为。
- 不引入第三方图表库。
- 不在账本页展示整月所有明细；账本页重点是选中日期的账单。

## 账本页设计

账本页顶部复用日记页同款日历结构：

- `年 / 月 / 周` 切换。
- 年视图选择月份后切回月视图。
- 月视图选择某日。
- 周视图选择某日。

选中日期后展示：

- 当月概览：收入、支出、结余。
- 当天账单列表。
- 当天无记录时显示“这天还没有账目”。

点击 `+` 新增账单时：

- 进入记账编辑页。
- 日期默认填充账本当前选中日期。

## 统计页设计

统计页顶部使用 `周账 / 月账 / 年账` 切换。

周账：

- 展示当前周标题。
- 支持上一周、下一周。
- 汇总本周收入、支出、结余。
- 展示 7 天趋势。
- 展示本周支出分类和收入分类排行。

月账：

- 展示当前月标题。
- 支持上一月、下一月。
- 汇总本月收入、支出、结余。
- 展示当月每日趋势。
- 展示本月支出分类和收入分类排行。

年账：

- 展示当前年标题。
- 支持上一年、下一年。
- 汇总全年收入、支出、结余。
- 展示 12 个月趋势。
- 展示全年支出分类和收入分类排行。

## 数据逻辑

在 `AccountingSummary.kt` 中补充：

- `recordsForWeek(records, selectedDate)`：按日历周筛选，周日到周六。
- `summaryForRecords(records)`：对任意记录集汇总收入、支出、结余。
- `dailyTotalsForDates(records, dates)`：生成每日趋势数据。
- 继续复用 `recordsForMonth`、`recordsForYear`、`monthlyTotalsForYear`、`categoryTotals`。

## 测试

- `recordsForWeek()` 正确返回周日到周六范围内记录。
- `dailyTotalsForDates()` 对无记录日期返回 0。
- `summaryForRecords()` 正确汇总任意记录集。
- 全量测试和 debug 构建必须通过。
